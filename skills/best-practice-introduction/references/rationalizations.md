# Rationalizations Reference

## Common Excuses and Rebuttals

| Excuse                         | Reality                                                                     |
| ------------------------------ | --------------------------------------------------------------------------- |
| "Apply everywhere immediately" | Immediate rollout causes chaos. Even urgent changes need support structure  |
| "No time for pilot phase"      | Pilot takes 1-2 weeks, saves months of failed adoption. Always worth time   |
| "Already paid for it"          | Sunk cost fallacy. Bad tools cost more in lost productivity than license    |
| "Mandate is non-negotiable"    | Question implementation approach, not decision. Phased respects mandate     |
| "Security can't wait"          | Security can have baseline exceptions and immediate enforcement on new code |
| "Team must adapt"              | Tools should adapt to teams or fail. Listen to feedback in pilot phase      |
| "Can't keep switching"         | One failed practice doesn't justify staying with second failed practice     |

## Closing Rationalizations

### "Must apply immediately everywhere"

**Response:** "Immediate rollout causes chaos. Even urgent changes need support structure."

**Action:** Propose immediate-with-baseline approach. Define support and communication plan.

### "No time for pilot phase"

**Response:** "Pilot takes 1-2 weeks, saves months of failed adoption."

**Action:** Offer compressed pilot option. Show cost of failed adoption.

### "Already paid for it"

**Response:** "Sunk cost fallacy. Bad tools cost more in lost productivity than license fees."

**Action:** Propose pilot of alternative. Frame decision on productivity, not cost.

### "Mandate is non-negotiable"

**Response:** "Question implementation approach, not decision. Phased rollout respects mandate."

**Action:** Propose how to execute mandate successfully. Define success criteria beyond compliance.

### "Team must adapt to the tool"

**Response:** "Tools should adapt to teams or fail. Listen to feedback in pilot phase."

**Action:** Propose feedback collection in pilot. Suggest tool configuration adjustments.
